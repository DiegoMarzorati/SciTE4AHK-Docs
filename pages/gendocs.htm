GenDocs 2.1

WARNING: GenDocs 3.0 will change the documentation syntax

GenDocs is a utility that allows you to automatically generate documentation for your functions. Using it is very simple: just add the following to the script and edit it for each function you want to document:

;
; Function: functionName
; Description:
;     Description (BBCode allowed)
; Syntax: Syntax(used, to, call [, this, function])
; Parameters:
;     Param1 - Description
;         More text
;         Even more text
;     Param2 - (Optional) BBCode is allowed too, like
;         [b]this[/b] or [code]this ; and comment highlighting[/code]
; Return Value:
;     Return value description (BBCode allowed)
; Remarks:
;     Remarks go here (BBCode allowed)
; Related: MyFunc1, MyFunc2, [bbcode]some literal text with commas (`,) and backticks (``). BBCode allowed.
;     When this parameter is omitted, it defaults to linking every
;     single documentated function in the file.
; Example:
;     An example or a pointer to a file or url goes here
;

Note: "Tab" in this help section means a tab and/or 3/4/5 spaces.

Special features

You can omit the Parameters, Return Value, Remarks, Related and Example sections.

When specifying example code or code in code tags you can use the following placeholder commands instead of writing code:

file:fileName  ; This represents the contents of the file fileName.
url:uri        ; This represents the contents of the web address uri.

There is a BBCode extension in order to create links to the AutoHotkey documentation:

[ahk]CommandName[/ahk]
[ahk=CommandName]Some text[/ahk]

The following BBCode extension adds a line break:

[br]

To escape square brackets [], commas (,) and backticks (`) in the Related section prepend a backtick to the character.

To force BBCode in an entry inside the Related section write [bbcode] before the text.

Running GenDocs

Open the script you want to generate the documentation for, then click on its icon in the toolbar () then on the Generate button.

Alternatively run GenDocs with the following command-line parameters:

GenDocs.ahk <scriptFile>

Documentation is always saved on the folder %SelectedScriptDir%\%SelectedScriptName%.

Example (Anchor by polyethene)

;
; Function: Anchor
; Description:
;     Defines how controls should be automatically positioned relatively to the new dimensions of a GUI when resized.
; Syntax: Anchor(cl[, a = "", r = 0])
; Parameters:
;     cl - a control HWND, associated variable name or ClassNN to operate on
;     a - (optional) one or more of the anchors: 'x', 'y', 'w' (width) and 'h' (height),
;         optionally followed by a relative factor, e.g. x h0.5
;     r - (OptIoNal) any SetWindowPos ([url]http://msdn2.microsoft.com/en-us/library/ms633545.aspx[/url]) flags,
;         SWP_NOZORDER is automatically applied to retain the current Z order
; Return Value:
;     None.
; Remarks:
;     I think none...
; Related: [bbcode]None.
; Example:
;     file:anchor-example.ahk
;
